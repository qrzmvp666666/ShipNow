## 概述

本节带你完成从零到本地运行的完整流程，包括依赖安装、Docker 服务启动、环境变量配置和数据库初始化。

## 所需工具

| 工具 | 本地开发 | 生产环境 | 说明 |
|------|:--------:|:--------:|------|
| Node.js ≥ 20 | ✅ | ✅ | JS 运行时 |
| pnpm ≥ 10 | ✅ | ✅ | 包管理器 |
| Docker | ✅ | — | 本地数据库和存储 |
| Git | ✅ | ✅ | 版本控制 |
| PostgreSQL | — | ✅ | 生产数据库（Neon/Supabase/Railway 等） |

## 克隆项目 & 安装依赖

```bash
git clone <your-repo-url> my-saas
cd my-saas
pnpm install
```

## 启动 Docker 本地服务

项目内置 `docker-compose.yml`，一键启动 **PostgreSQL** 和 **MinIO**（S3 兼容对象存储）：

```bash
docker compose up -d
```

| 服务 | 地址 | 说明 |
|------|------|------|
| PostgreSQL | `localhost:5432` | 用户名/密码均为 `postgres` |
| MinIO S3 API | `localhost:9000` | 对象存储接口 |
| MinIO 控制台 | `localhost:9001` | 账号 `minioadmin` / `minioadmin` |

> MinIO 启动时会自动创建 `avatars` bucket 并设置公开读权限。

## 基于模板创建独立项目

> 用同一模板开发多个项目时，需要为每个项目配置独立的 Docker 服务，其余步骤与上方标准流程完全相同。

克隆项目后，在执行 `docker compose up -d` **之前**，先修改以下两个文件：

### 1. docker-compose.yml

将容器名、端口、volume 名改为项目专属名称，避免与其他项目冲突：

```yaml
services:
  postgres:
    container_name: myproject-postgres      # 改成你的项目名
    ports:
      - "5433:5432"                         # 换一个未被占用的端口
    volumes:
      - myproject_postgres_data:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: myproject                # 可选：改数据库名

  minio:
    container_name: myproject-minio
    ports:
      - "9002:9000"
      - "9003:9001"
    volumes:
      - myproject_minio_data:/data

  minio-setup:
    container_name: myproject-minio-setup

volumes:
  myproject_postgres_data:
  myproject_minio_data:
```

### 2. .env

```dotenv
DATABASE_URL="postgresql://postgres:postgres@localhost:5433/myproject"
S3_ENDPOINT="http://localhost:9002"
```

改完后按标准流程继续即可，两个项目可同时运行互不干扰。

## 配置环境变量

```bash
cp .env.local.example .env
```

本地开发最小配置：

```dotenv
# 数据库（Docker）
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/supastarter"

# 应用 URL
NEXT_PUBLIC_MARKETING_URL="http://localhost:3001"
NEXT_PUBLIC_SAAS_URL="http://localhost:3000"
NEXT_PUBLIC_DOCS_URL="http://localhost:3002"

# Auth 密钥
BETTER_AUTH_SECRET="your-random-secret-here"

# 邮件（本地打印到终端，无需真实服务）
MAIL_FROM="noreply@localhost"
CONTACT_EMAIL="admin@localhost"

# 存储（Docker MinIO）
S3_ACCESS_KEY_ID="minioadmin"
S3_SECRET_ACCESS_KEY="minioadmin"
S3_ENDPOINT="http://localhost:9000"
S3_REGION="us-east-1"
NEXT_PUBLIC_AVATARS_BUCKET_NAME="avatars"
```
